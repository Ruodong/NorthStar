# SpecKit Tasks to Issues

Converts SpecKit task list into GitLab issues for external tracking.

## Flow

1. Read the task list generated by `/speckit-tasks`
2. For each task, create a GitLab issue with:
   - **Title** — task title
   - **Description** — task description + acceptance criteria + file list
   - **Labels** — `speckit`, feature name, `backend`/`frontend`/`schema` as applicable
   - **Milestone** — if specified by user
3. Optionally create a parent epic or tracking issue that links all sub-issues
4. Report created issue URLs to user

## Rules

- Confirm with user before creating issues (this is an external action)
- Use `gh` or GitLab API via the remote server (local cannot reach gitlab.xpaas.lenovo.com)
- Include spec reference (FR/NFR IDs) in issue body
- Don't create duplicate issues — check existing issues first
